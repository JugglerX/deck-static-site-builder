Paradigm is a flexible css framework that belives in

Media Query Helpers

Media queries are baked into the grid. Change breakpoints with a global variable. Convienent helpers can be used anywhere in the css for specific overrides.


h1 {
  font-size: 36px;
  @media @mobile {
    font-size: 20px; 
  }
}

Flexible Grid

Flexible grid system. No nested containers, No multiple classes. 1 .column() mixin, you can use anywhere. It's surprsingly zen.


.portfolio {
  .column(4);
  .last(3);
}

No Markup

Styles are applied as mixins in the css. Sometimes you can't change the markup. Enforces consistency.

Components, View Modes & Globals

Features and components get there own css. Context changes to components are handled with view modes. Things like buttons, forms and other common elements are globals and are applied as classes in the markup or as exetensions in css. It's a stylistic take on OOP or BEM methodology (if your into that). It's stragnly explicit, maintainable and practically reusable.


<div class="portfolio">
  <img class="portfolio-image" src="images/portfolio-world-surfaris.jpg" />
  <h3>World Surfaris</h3>
  <p class="portfolio-type">Australia's Largest Render Manufacturer</p>        
  <p>Drupal architecture, design and development, custom theme, custom modules, e-commerce</p>
</div>
